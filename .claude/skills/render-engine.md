# render-engine

## Mission

Thin yt-channel-specific shim around the **global `video-editor` skill**. Builds the `editing/` folder from the run's artifacts and hands off. The generic skill does all the actual rendering, sub-agent routing, and delivery encoding.

## Inputs

- `run_id` (required)
- `script_path`: `runs/<id>/script.json`
- `assets_manifest`: `runs/<id>/assets/manifest.json`

## Process

1. **Build the `editing/` folder** for this run:
   ```
   runs/<id>/editing/
   ├── brief.md          ← generated from script.json + niches.formula_json
   ├── assets/
   │   ├── clips/        ← symlinks to runs/<id>/assets/scene_N/video.mp4
   │   ├── images/       ← symlinks to assets/scene_N/image.png
   │   └── audio/        ← (empty for MVP — text-overlay-only)
   ├── style/
   │   └── brand.json    ← copied from <workspaceDir>/config/brand-default.json
   └── output/           ← (empty, video-editor writes here)
   ```

2. **Generate `brief.md`** from script.json:
   ```yaml
   ---
   preset: short
   duration_sec: <from script>
   aspect: 9:16
   output_path: editing/output/final.mp4
   voice: none
   captions: none
   delivery: youtube
   style_anchor: editing/style/brand.json
   ---
   # Intent
   <from niches.formula_json.thesis or script-smith intent>

   # Beats
   <one line per scene from script.json scenes[]>
   ```

3. **Invoke `video-editor`**:
   ```
   Use the video-editor skill with arg "runs/<id>/editing/"
   ```

   Or via Skill tool:
   ```
   Skill({skill: "video-editor", args: "runs/<id>/editing/"})
   ```

4. **Verify output** at `runs/<id>/editing/output/final.mp4`. Move/symlink to `runs/<id>/final.mp4` for downstream stages.

5. Update SQLite `runs.output_path` + emit `render_engine:finished`.

## Output

- `runs/<id>/final.mp4` (1080×1920, 30fps, h264, ~8 Mbps per brand.json delivery.youtube_shorts spec)
- `runs/<id>/editing/output/SUMMARY.md` (auto-generated by video-editor)
- `runs/<id>/editing/work/routing.json` (which sub-agents the video-editor used, if any)

## Constraints

- DO NOT reimplement rendering logic in this agent — always delegate to video-editor.
- If video-editor reports issues (missing assets, conflicting style), surface them to the orchestrator before failing.
- For yt-channel, the brand.json lives at `config/brand-default.json` and defines the Shorts-specific style (red accent, Impact font, bottom-center captions, etc.)
- Budget: ~3 minutes per Short render (Remotion 60-180s + encoding 20s).

## Why this delegation

The generic video-editor is reusable across:
- yt-channel (this project) — Shorts
- Zephly templates — explainers and product demos
- NoCode18 — client video ads
- Any future project that needs video composition

Keeping rendering logic in the generic skill avoids duplication and lets the team specialize once (sub-agents like vfx-artist, motion-designer) for all projects.

## Files

- Reads: `runs/<id>/script.json`, `runs/<id>/assets/`, `config/brand-default.json`
- Writes: `runs/<id>/editing/brief.md`, `runs/<id>/final.mp4`
- Delegates: ALL actual rendering to `~/.claude/skills/video-editor/`
